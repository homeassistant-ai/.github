### Code Quality
- Follow existing code patterns and style in each project
- Add tests for new features or bug fixes
- Ensure all automated tests pass before submitting PRs
- Run linters and type checkers as specified in project docs

### Communication
- Use issue templates when reporting bugs or requesting features
- Be clear and concise in issue descriptions and PR explanations
- Respond to feedback on PRs in a timely manner
- Be respectful and constructive in all interactions

### Pull Requests
- Keep PRs focused on a single issue or feature
- Write clear commit messages
- Update documentation for user-facing changes
- Link to related issues using `Fixes #123` or `Closes #123`
